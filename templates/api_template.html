Our API!

Here are some information about how to use our API.

Authentication

Rate Yourself API doesn't use any authentication and is free to use for those who wish. Note though that we make use of the API provided from GitHub and it only allow 60 requests per hour so we have that limitation.

Errors

Rate Yourself API make use of the standard HTTP Status Codes. The following errors can be shown:

400 if email or stackid parametres are missing in GET request.
If either github email or stackoverflow id are invalid no data from that account will be returned.

Endpoints

All requests are sent to: {{address}}

Make HTTP GET request to:
/api/v1/profile?email=value&stackid=value

Email is the registered email on github.com and stackid is the account id on stackoverflow.com which can be found in the url when logged in.

The response will be a JSON and you will get the following information:
     email: the registered email on github.com.
     gbio: the users github biography.
     gcreated: date for creation of github account.
     gfollowers: number of followers on github.com.
     gpublic_repos: number of public repositories on github.com.
     gupdated: when the github account was last updated.
     hireable: whether the user is hireable or not.
     level: your Rate Yourself level.
     location: the users location.
     name: name of the user.
     picture: avatar picture from github.com.
     sbronze_badge: number of bronze badges on stackoverflow.
     score: your Rate Yourself score.
     sgold_badge: number of gold badges on stackoverflow.
     sreputation: the stackoverflow reputation.
     ssilver_badge: number of silver badges on stackoverflow.
     stackoverflow_link: link to account on stackoverflow.
     stars: the total number of star on public repositories from the github account.

Examples

Example request: http://127.0.0.1:5000/api/v1/profile?email=annika.h.magnusson@gmail.com&stackid=4632228
